---
title: Comprehensive mode
description: Comprehensive mode response structure with detailed analysis fields
---

Comprehensive mode provides rich, detailed analysis including timelines, direct quotes, multiple perspectives, and verification status. This mode uses approximately 4,000 tokens per request and responses are cached for 30 minutes.

## Response structure

Comprehensive mode includes all basic mode fields plus extensive analysis components.

### Base fields

<ResponseField name="query" type="string" required>
  The original search query that was executed
</ResponseField>

<ResponseField name="analysis_mode" type="string" required>
  Set to `"comprehensive"` for comprehensive mode responses
</ResponseField>

### Analysis fields

<ResponseField name="comprehensive_analysis" type="string" required>
  A detailed 500+ word analysis providing deep context, background, and implications. Includes specific details, dates, numbers, and concrete information.
</ResponseField>

<ResponseField name="key_findings" type="array" required>
  Array of categorized findings with confidence levels
  
  <Expandable title="Key finding object properties">
    <ResponseField name="category" type="string" required>
      Finding category: `"main_story"`, `"development"`, `"context"`, or `"impact"`
    </ResponseField>
    
    <ResponseField name="title" type="string" required>
      Specific finding title
    </ResponseField>
    
    <ResponseField name="content" type="string" required>
      Detailed explanation with specifics, numbers, and dates
    </ResponseField>
    
    <ResponseField name="sources" type="array" required>
      Array of source URLs supporting this finding
    </ResponseField>
    
    <ResponseField name="confidence" type="string" required>
      Confidence level: `"high"`, `"medium"`, or `"low"`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="timeline" type="array" required>
  Chronological sequence of events with significance
  
  <Expandable title="Timeline event properties">
    <ResponseField name="date" type="string" required>
      Event date in YYYY-MM-DD format
    </ResponseField>
    
    <ResponseField name="event" type="string" required>
      Specific event description
    </ResponseField>
    
    <ResponseField name="source" type="string" required>
      Source name for this event
    </ResponseField>
    
    <ResponseField name="significance" type="string" required>
      Why this event matters in the broader context
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="direct_quotes" type="array" required>
  Array of exact quotes with full attribution
  
  <Expandable title="Quote object properties">
    <ResponseField name="quote" type="string" required>
      Exact quote from the source
    </ResponseField>
    
    <ResponseField name="speaker" type="string" required>
      Speaker name and title
    </ResponseField>
    
    <ResponseField name="context" type="string" required>
      When and where this was said
    </ResponseField>
    
    <ResponseField name="source_url" type="string" required>
      URL of the source containing the quote
    </ResponseField>
    
    <ResponseField name="significance" type="string" required>
      Why this quote is important
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="related_context" type="string" required>
  Historical background, related events, and broader context that helps understand the topic
</ResponseField>

<ResponseField name="multiple_perspectives" type="array" required>
  Analysis from different stakeholder viewpoints
  
  <Expandable title="Perspective object properties">
    <ResponseField name="viewpoint" type="string" required>
      Perspective group (e.g., "Supporters", "Critics", "Experts")
    </ResponseField>
    
    <ResponseField name="content" type="string" required>
      What this group thinks or says about the topic
    </ResponseField>
    
    <ResponseField name="sources" type="array" required>
      Array of source URLs for this perspective
    </ResponseField>
    
    <ResponseField name="reasoning" type="string" required>
      Why this group holds this position
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="implications" type="object" required>
  Impact analysis covering short-term and long-term effects
  
  <Expandable title="Implications object properties">
    <ResponseField name="short_term" type="string" required>
      Immediate consequences and effects
    </ResponseField>
    
    <ResponseField name="long_term" type="string" required>
      Potential long-term impacts and outcomes
    </ResponseField>
    
    <ResponseField name="stakeholders_affected" type="array" required>
      Array of stakeholder groups affected by this topic
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="verification_status" type="object" required>
  Fact verification and claim tracking
  
  <Expandable title="Verification status properties">
    <ResponseField name="confirmed_facts" type="array" required>
      Array of verified information with sources
    </ResponseField>
    
    <ResponseField name="unconfirmed_claims" type="array" required>
      Array of claims that need verification
    </ResponseField>
    
    <ResponseField name="contradictory_information" type="array" required>
      Array of conflicting reports or information
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="raw_results" type="array" required>
  Array of raw search results with relevance scores
  
  <Expandable title="Raw result properties">
    <ResponseField name="title" type="string" required>
      Source article title
    </ResponseField>
    
    <ResponseField name="snippet" type="string" required>
      Brief description
    </ResponseField>
    
    <ResponseField name="url" type="string" required>
      Source URL
    </ResponseField>
    
    <ResponseField name="relevance_score" type="number" required>
      Relevance score on a 1-10 scale
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="summary" type="string" required>
  Executive summary of the entire analysis
</ResponseField>

<ResponseField name="total_results" type="number" required>
  Total number of raw results in the response
</ResponseField>

<ResponseField name="search_time" type="string" required>
  ISO 8601 timestamp of when the search was executed
</ResponseField>

<ResponseField name="source" type="string" required>
  Set to `"grok-comprehensive-analysis"` for comprehensive mode responses
</ResponseField>

<ResponseField name="citations" type="array">
  Array of citation URLs from all sources
</ResponseField>

<ResponseField name="citation_metadata" type="array">
  Array of metadata objects for each citation
</ResponseField>

## Usage notes

- Comprehensive analyses are cached for 30 minutes to optimize performance and reduce API costs
- This mode uses approximately 4,000 tokens per request (double the basic mode)
- The analysis includes specific extraction of dates, numbers, names, and locations
- Direct quotes include full attribution with speaker, context, and significance
- Timeline events are sorted chronologically with significance explanation
- Multiple perspectives analyze the topic from different stakeholder viewpoints
- Verification status helps distinguish confirmed facts from unconfirmed claims
- Best suited for complex topics requiring deep contextual understanding
