This guide will help you configure and start using the Grok Search MCP Server with Claude Desktop.

## Overview

You'll complete these steps:

1. Get an xAI API key
2. Configure Claude Desktop
3. Restart Claude Desktop
4. Make your first search

## Get your xAI API key

<Steps>
  <Step title="Visit the xAI Developer Portal">
    Navigate to [console.x.ai](https://console.x.ai/) in your web browser.
  </Step>
  
  <Step title="Create an account or sign in">
    If you don't have an account, create one. Otherwise, sign in with your existing credentials.
  </Step>
  
  <Step title="Generate an API key">
    1. Navigate to the API Keys section
    2. Click "Create new API key" or similar button
    3. Give your key a descriptive name (e.g., "Grok MCP Server")
    4. Copy the generated API key
    
    <Warning>
    The API key format starts with `xai-`. Store it securely - you won't be able to view it again.
    </Warning>
  </Step>
</Steps>

## Configure Claude Desktop

Now configure Claude Desktop to use the Grok Search MCP Server.

<Steps>
  <Step title="Locate the configuration file">
    Find your Claude Desktop configuration file based on your operating system:
    
    <CodeGroup>
    ```bash macOS
    ~/Library/Application Support/Claude/claude_desktop_config.json
    ```
    
    ```bash Windows
    %APPDATA%/Claude/claude_desktop_config.json
    ```
    
    ```bash Linux
    ~/.config/Claude/claude_desktop_config.json
    ```
    </CodeGroup>
    
    <Note>
    If the file doesn't exist, create it. Make sure the parent directories exist.
    </Note>
  </Step>
  
  <Step title="Edit the configuration file">
    Open the configuration file in your preferred text editor:
    
    <CodeGroup>
    ```bash macOS
    open -e ~/Library/Application\ Support/Claude/claude_desktop_config.json
    ```
    
    ```bash Windows (PowerShell)
    notepad $env:APPDATA\Claude\claude_desktop_config.json
    ```
    
    ```bash Linux
    nano ~/.config/Claude/claude_desktop_config.json
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Add the server configuration">
    Add the Grok Search MCP Server configuration. Replace `your-api-key-here` with your actual xAI API key:
    
    ```json
    {
      "mcpServers": {
        "grok-search": {
          "command": "npx",
          "args": ["grok-search-mcp"],
          "env": {
            "XAI_API_KEY": "your-api-key-here"
          }
        }
      }
    }
    ```
    
    <Warning>
    Ensure the JSON is valid. Missing commas or brackets will prevent Claude Desktop from starting.
    </Warning>
    
    **Advanced configuration** with custom timeout and retry settings:
    
    ```json
    {
      "mcpServers": {
        "grok-search": {
          "command": "npx",
          "args": ["grok-search-mcp"],
          "env": {
            "XAI_API_KEY": "your-api-key-here",
            "GROK_TIMEOUT": "45000",
            "GROK_MAX_RETRIES": "5"
          }
        }
      }
    }
    ```
    
    Environment variables:
    - `XAI_API_KEY` (required): Your xAI API key
    - `GROK_TIMEOUT` (optional): Request timeout in milliseconds (default: 30000)
    - `GROK_MAX_RETRIES` (optional): Maximum retry attempts (default: 3)
  </Step>
  
  <Step title="Save the configuration file">
    Save the file and ensure there are no syntax errors.
    
    **Validate your JSON** (optional):
    
    <CodeGroup>
    ```bash macOS/Linux
    python -m json.tool ~/Library/Application\ Support/Claude/claude_desktop_config.json
    ```
    
    ```bash Windows (PowerShell)
    Get-Content $env:APPDATA\Claude\claude_desktop_config.json | ConvertFrom-Json
    ```
    </CodeGroup>
  </Step>
</Steps>

## Restart Claude Desktop

<Steps>
  <Step title="Quit Claude Desktop completely">
    Make sure Claude Desktop is fully closed (not just minimized):
    
    - **macOS**: Press `Cmd+Q` or right-click the dock icon and select "Quit"
    - **Windows**: Right-click the system tray icon and select "Exit"
    - **Linux**: Use your system's application quit method
  </Step>
  
  <Step title="Start Claude Desktop">
    Launch Claude Desktop again. The server will automatically connect on startup.
  </Step>
  
  <Step title="Verify the server is connected">
    In Claude Desktop, you should see the Grok Search tools available. Look for tools like:
    - `grok_search`
    - `grok_web_search`
    - `grok_news_search`
    - `grok_twitter`
    - `health_check`
    
    <Note>
    If you don't see the tools, check the Claude Desktop logs or verify your configuration file.
    </Note>
  </Step>
</Steps>

## Make your first search

Now you're ready to perform searches using Grok!

<Steps>
  <Step title="Test with a basic search">
    In Claude Desktop, ask:
    
    > "Search for the latest developments in artificial intelligence"
    
    Claude will use the `grok_search` tool to fetch current information.
  </Step>
  
  <Step title="Try a comprehensive analysis">
    For detailed analysis with timelines and multiple perspectives, ask:
    
    > "Give me a comprehensive analysis of recent climate change policy developments"
    
    This will use comprehensive mode, providing:
    - Detailed timeline of events
    - Direct quotes from sources
    - Multiple perspectives
    - Historical context
    - Implications analysis
  </Step>
  
  <Step title="Search with date filters">
    For time-specific searches, ask:
    
    > "Search for news about quantum computing breakthroughs from January 2025 to March 2025"
    
    The server will filter results to the specified date range.
  </Step>
  
  <Step title="Check server health">
    Verify the server is working properly:
    
    > "Check the health status of the Grok search server"
    
    This returns diagnostic information including:
    - Server health status
    - API connectivity
    - Request statistics
    - Cache size
    - Success rate
  </Step>
</Steps>

## Troubleshooting

### Server not appearing in Claude Desktop

1. Verify your configuration file has valid JSON syntax
2. Check that Claude Desktop was completely restarted
3. Ensure the `XAI_API_KEY` is set correctly
4. Check Claude Desktop logs for error messages

### API key errors

**Error**: "API service is not healthy - missing XAI_API_KEY"

**Solution**:
1. Verify the API key is correctly set in `claude_desktop_config.json`
2. Ensure there are no extra spaces or quotes around the key
3. Confirm the key starts with `xai-`

### Timeout errors

**Error**: "Request timeout after 30000ms"

**Solution**:
1. Increase the `GROK_TIMEOUT` environment variable to `45000` or higher
2. Check your internet connection
3. Try using basic mode instead of comprehensive mode for faster responses

### No results returned

**Possible causes**:
1. Query may be too specific or too broad
2. Date range may be too narrow
3. Topic may have limited online coverage

**Solution**: Try rephrasing the query or adjusting date filters
